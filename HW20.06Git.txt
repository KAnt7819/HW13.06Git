1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)? 
2.Как удалить все новые файлы, если их не нужно включать в коммит? (Очистка репозитория от лишних файлов)
3.Как вернуть файл из индекса в рабочий каталог? 
4.Как отменить ошибочный commit ? (перечислить все варианты).
5.Как исправить комментарии к коммиту?
6.Как исправить коммит отправленный на GitHub?
7.Почему нельзя применять команду git reset к коммитам которые выгружены на сервер Git?
В ответах на вопросы 1-6  необходимо указать конкретные команда git. 


1. Чтобы отказаться от изменений в файле или файлах в рабочем каталоге, используем команду git checkout -- <file> для каждого файла, который хотим отменить.

2. Чтобы удалить все новые файлы, которые не должны быть включены в коммит, используем команду git clean -f. Она удалит все новые, неотслеживаемые файлы в рабочем каталоге. Нужно только быть уверенным в необходимости удаления этих файлов, так как она безвозвратно удалит файлы.

3. Чтобы вернуть файл из индекса в рабочий каталог, используем команду git reset HEAD <file>. 

4. Чтобы отменить ошибочный коммит, можно использовать команду git revert <commit>. Она создаст новый коммит, который отменяет изменения, внесенные указанным коммитом. Или  воспользоваться командой git reset, но так как она изменяет историю коммитов, и может потребоваться принудительная выгрузка (force push) для удаления коммитов с сервера, нужно убедиться в необходимости этой процедуры.

5. Чтобы исправить комментарии к последнему коммиту, используем команду git commit --amend. Она позволяет внести изменения в последний коммит, включая его комментарий.

6. Чтобы исправить коммит, отправленный на GitHub, необходимо внести изменения локально, используя команды, описанные выше, и затем выполнить принудительную выгрузку (force push) изменений на GitHub с помощью команды git push -f.

7. Команда git reset используется для изменения истории коммитов, возвращая указатель HEAD и веток к определенному коммиту. Если применить git reset к коммитам, которые уже выгружены на сервер Git, это может привести к несоответствиям истории коммитов между локальным репозиторием и удаленным репозиторием. Другие разработчики, работающие с удаленным репозиторием, могут иметь разные версии истории, и сбой команды git reset
